# 统一实验规范

## 1. 目的

本文件定义 FedJD 全部实验阶段共享的协议。所有阶段文档都默认继承本规范，除非该阶段文档明确覆盖。

## 2. 实验命名规范

实验 ID 统一采用：

`阶段-数据集-模型-目标数m-客户端数K-参与率C-本地步E-版本标签`

示例：

`S1-synth_m2-mlp-m2-K8-C0.5-E1-fulljac-v1`

## 3. 固定记录项

每次实验必须记录：
- 代码版本或提交号
- 随机种子
- 数据划分方式
- 客户端数 `K`
- 参与率 `C`
- 本地步数 `E`
- batch size
- 学习率
- 目标数 `m`
- 模型参数量 `d`
- Jacobian 上传格式
- 是否压缩通信
- 单轮上行字节数
- 单轮下行字节数
- 单轮 wall-clock time
- 峰值显存或内存
- 每轮目标值
- 最终 checkpoint 路径

## 4. 对照原则

所有对照实验默认保持以下条件一致：
- 相同数据划分
- 相同模型结构
- 相同参与率
- 相同本地训练 budget
- 相同训练轮数或相同总计算预算

如不能保持一致，必须在实验说明里明确写出原因。

## 5. 随机性控制

每项主实验至少使用 3 个随机种子。正式结果表默认报告：
- mean
- std
- 最优与最差 seed 的差异摘要

## 6. 默认指标

### 优化指标
- 每个目标的最终值
- 每个目标的训练曲线
- Pareto gap 或 stationarity gap
- Hypervolume（若适用）

### 联邦系统指标
- 每轮上行通信量
- 每轮下行通信量
- 总通信量
- 单轮训练时间
- 总训练时间
- 峰值 GPU/CPU 内存

### 稳定性指标
- 训练是否发散
- NaN/Inf 次数
- 方向范数
- Jacobian Frobenius norm

## 7. 结果保存格式

每个实验运行目录建议至少包含：
- `config.yaml`
- `metrics.csv`
- `summary.md`
- `stdout.log`
- `plots/`
- `checkpoints/`

## 8. 放行标准

一个阶段进入下一阶段前，至少满足：
- 指标采集稳定
- 实验可复现
- 至少一个基线配置能稳定收敛
- 关键 bug 已闭环，不依赖人工临场修补

## 9. 不允许的捷径

- 不允许只给单次运行结果就下结论
- 不允许在不同方法间使用不同数据划分却不说明
- 不允许只报告最好 seed
- 不允许压缩方法改变总训练预算却仍当作公平对比

## 10. 当前研究假设

当前主线默认：
- 全局固定 `m` 个目标
- 客户端只是数据承载者
- 服务器基于聚合后的全局 Jacobian 选择方向
- 部分参与为默认设置，全参与作为特例保留
- 单机模拟联邦轮次，后续再扩展真实分布式
