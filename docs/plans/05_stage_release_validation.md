# 阶段 5：封版验证、消融与开源准备

## 1. 阶段目标

本阶段不是再找新点子，而是为最终论文和开源版本建立可信证据链。

## 2. 必做验证

### 2.1 复现验证
- 主实验至少重跑 3 次
- 关键表格与图可从原始日志重新生成

### 2.2 消融验证
至少包括：
- 不同聚合器
- 不同 Jacobian 估计方式
- 不同参与率
- 不同本地步数
- 不同压缩方案

### 2.3 鲁棒性验证
- 不同 seed
- 不同 non-IID 划分
- 不同模型大小
- 轻微超参数扰动

### 2.4 失败案例整理
必须主动记录：
- 哪些配置发散
- 哪些场景下 FedJD 不占优
- 哪些压缩方案破坏了方法语义

## 3. 论文输出导向

本阶段每个实验都要能服务于论文中的某一段结论。不能再跑“没有明确用途”的实验。

建议按章节对应：
- 主表支撑主结论
- 消融图支撑设计选择
- 通信图支撑工程可行性
- 失败案例支撑讨论与局限性

## 4. 开源输出导向

准备以下内容：
- 统一运行入口
- 默认配置文件
- 复现实验说明
- 结果目录结构说明
- 常见问题说明

## 5. 最终放行清单

全部满足才可视为封版：
- 主结果可复现
- 消融完整
- 图表与原始日志一致
- 方法边界写清楚
- 失败案例没有被选择性隐藏
- README 可指导他人复现最小实验

## 6. 阶段交付物

- 最终实验报告
- 论文图表草稿
- 复现清单
- 开源前检查表

## 7. 最终要回答的问题

- FedJD 是否成立为一个独立的方法，而不是 FMGDA 的变体
- 它的优势场景是什么
- 它的主要代价是什么
- 通过哪些压缩和工程策略，代价变得可接受
