# NFJD (Normalized Jacobian Descent) 方法更新说明

## 1. 方法概述

NFJD 是一种联邦学习多目标优化算法，通过目标对齐分数机制，在保持低通信开销的同时，实现多个任务的平衡优化。

## 2. 核心创新点

### 2.1 目标对齐分数（Goal Alignment Scores）
- **定义**：每个目标的梯度与本地更新方向的内积，衡量更新对各目标的"友好程度"
- **计算**：`align_scores = G @ Δθ`，其中 G 是 Jacobian 矩阵，Δθ 是本地更新量
- **作用**：评估本地更新对每个目标的影响方向和程度

### 2.2 轻量级信息传输
- **传输内容**：仅传输 m 个浮点数（每个目标一个对齐分数）
- **通信开销**：极低，与任务数 m 线性相关
- **隐私保护**：不传输完整 Jacobian 矩阵，降低隐私风险

### 2.3 动态权重调整
- **全局对齐分数**：计算每个目标在所有客户端上的平均对齐分数 `global_align`
- **权重调整**：根据 `global_align` 调整客户端权重，优先考虑对所有任务都友好的客户端
- **任务保护**：对被普遍伤害的目标进行补偿性权重调整

### 2.4 多目标健康监控
- **监控指标**：`global_align` 实时监控各目标的优化状况
- **早期预警**：为任务退化提供早期预警机制
- **可视化**：可通过 `global_align` 曲线分析多目标优化过程

## 3. 算法流程

### 3.1 服务器端流程
1. **初始化**：
   - 初始化全局模型
   - 初始化 `GlobalMomentum`
   - 设置超参数

2. **客户端采样**：
   - 根据 `participation_rate` 采样客户端

3. **模型分发**：
   - 将全局模型发送给采样的客户端

4. **接收客户端结果**：
   - 接收每个客户端的 `delta_theta` 和 `align_scores`
   - 计算每个客户端的样本权重

5. **计算全局对齐分数**：
   - 对每个目标，计算所有客户端 `align_scores` 的加权平均
   - 得到 `global_align` 向量

6. **调整客户端权重**：
   - 根据 `global_align` 调整客户端的聚合权重
   - 对被普遍伤害的目标，增加对其友好的客户端权重

7. **聚合更新**：
   - 基于调整后的权重聚合 `delta_theta`
   - 应用全局动量
   - 更新全局模型

8. **评估与统计**：
   - 评估全局模型性能
   - 记录 `global_align` 等统计信息
   - 检查任务退化情况

### 3.2 客户端流程
1. **接收模型**：
   - 接收服务器发送的全局模型

2. **本地更新**：
   - **前向传播**：计算模型输出和每个任务的损失
   - **Jacobian 计算**：每 `recompute_interval` 步计算 Jacobian 矩阵
   - **MinNorm 求解**：求解 MinNorm 问题得到最优方向
   - **自适应缩放**：调整方向范数
   - **动量更新**：应用冲突感知动量
   - **参数更新**：更新本地模型参数

3. **计算对齐分数**：
   - 获取计算的 Jacobian 矩阵 G
   - 计算本地更新量 Δθ
   - 计算 `align_scores = G @ Δθ`

4. **上传结果**：
   - 上传 `delta_theta` 和 `align_scores` 到服务器

## 4. 关键参数

| 参数 | 描述 | 默认值 |
|------|------|--------|
| `conflict_aware_momentum` | 是否启用冲突感知动量 | True |
| `rescaling_max_scale` | 自适应缩放的最大缩放因子 | 5.0 |
| `recompute_interval` | Jacobian 重算间隔 | 4 |
| `local_epochs` | 本地训练轮数 | 3 |
| `learning_rate` | 学习率 | 0.01 |
| `participation_rate` | 客户端参与率 | 0.5 |

## 5. 通信开销分析

| 传输内容 | 大小 | 说明 |
|---------|------|------|
| `delta_theta` | n | 参数更新量（n 是参数数） |
| `align_scores` | m | 目标对齐分数（m 是任务数） |
| 总通信量 | n + m | 与传统方法相比显著减少 |

## 6. 隐私保护分析

- **Jacobian 矩阵**：不上传完整 Jacobian 矩阵，降低隐私风险
- **对齐分数**：仅传输标量值，不包含原始梯度信息
- **安全性**：可与差分隐私、安全聚合等技术结合使用

## 7. 实验设置

### 7.1 MultiMNIST 数据集
- **任务**：识别左右两个数字（2个任务）
- **数据分割**：IID 和 Non-IID（Dirichlet 分布）
- **客户端数**：10
- **参与率**：50%

### 7.2 CelebA 数据集
- **任务**：预测多个属性（4个任务）
- **数据分割**：IID 和 Non-IID（Dirichlet 分布）
- **客户端数**：10
- **参与率**：50%

## 8. 预期效果

- **多目标优化**：保持 NFJD 在多目标优化上的优势
- **通信效率**：显著降低通信开销
- **隐私保护**：提高隐私安全性
- **Non-IID 适应**：缓解 Non-IID 下的任务退化
- **收敛稳定性**：通过冲突感知动量提高稳定性
